---
title: "Preview post per platform"
api: "GET /posts/{post}/preview"
description: "Preview how a post will render on each enabled platform — applies platform-specific sanitization (length truncation, forbidden chars, etc.) without publishing."
---

## Path parameters

<ParamField path="post" type="string" required>
  The UUID of the post.
</ParamField>

## Behaviour

- The post must belong to the caller's current workspace; otherwise the response is `404`.
- Returns the original content alongside per-platform sanitized versions and length statistics.
- `truncated` is `true` when the platform's `max_content_length` forced the content to be shortened.

## Response

<ResponseExample>

</ResponseExample>

</RequestExample>
