[Perplexity](https://www.perplexity.ai) supports custom remote MCP connectors, so you can connect TryPost and manage social media scheduling from Perplexity chat and [Computer](https://www.perplexity.ai/changelog/what-we-shipped---march-13-2026) workflows.

<Note>
  This guide connects **TryPost's MCP server into Perplexity**. Perplexity also publishes its own search MCP for use *inside* other clients (Claude, Cursor, etc.) — that is a different product documented at [Perplexity MCP Server](https://docs.perplexity.ai/docs/getting-started/integrations/mcp-server).
</Note>

## Prerequisites

- A Perplexity **Pro, Max, or Enterprise** account (custom remote connectors are not available on Free — see [Adding custom remote connectors](https://www.perplexity.ai/help-center/en/articles/13915507-adding-custom-remote-connectors))
- A TryPost Cloud account with an active trial or subscription ([start a 7-day free trial](/knowledge-base/plans-and-billing))

## Setup

<Steps>
  <Step title="Open Connectors">
    In Perplexity, open **Account Settings → Connectors** (from Computer you can also open **Connectors** in the sidebar). Click **+ Custom connector**.
  </Step>
  <Step title="Add a remote connector">
    Select **Remote** and fill in:

    | Field | Value |
    |-------|-------|
    | **Name** | `TryPost` |
    | **MCP Server URL** | `https://app.trypost.it/mcp/trypost` |
    | **Authentication** | **OAuth 2.0** (leave Client ID / Secret empty — TryPost uses Dynamic Client Registration) |
    | **Transport** | Streamable HTTP (usually auto-detected; prefer this over SSE) |

    Acknowledge the custom-connector risk notice, then click **Add**.
  </Step>
  <Step title="Authorize TryPost">
    Open the new connector card and complete the OAuth sign-in. Sign in to TryPost and click **Allow**. The connector should show as connected with TryPost's tools available.
  </Step>
  <Step title="Enable TryPost in a thread">
    Start a new conversation (or Computer task), enable the **TryPost** connector for that thread, and ask:

    ```
    List all my scheduled posts
    ```

    Perplexity will call the TryPost tools to fetch your posts.
  </Step>
</Steps>

<Note>
  Perplexity reaches the MCP server from its own cloud over HTTPS. The hosted endpoint `https://app.trypost.it/mcp/trypost` works out of the box. Self-hosted instances must be publicly reachable over HTTPS (or use a tunnel).
</Note>

<Accordion title="API key authentication instead of OAuth">
  If OAuth isn't available in your Perplexity UI, choose **API Key** authentication and paste a TryPost Personal Access Token from **Settings &gt; API Keys** on the [TryPost dashboard](https://app.trypost.it). Prefer OAuth when both options work.
</Accordion>

<Accordion title="Enterprise / organization connectors">
  On Enterprise, admins can share connectors organization-wide and control whether members may add their own. Members still authenticate individually when the connector uses OAuth, unless your admin configures org-wide auth.
</Accordion>
