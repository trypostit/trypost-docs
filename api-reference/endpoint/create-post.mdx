---
title: "Create post"
api: "POST /posts"
description: "Create a new post targeting one or more social accounts. Returns the created post with status 201."
---

## Request

<ParamField body="platforms" type="array" required>
  At least one platform entry. Each entry pins this post to a connected social account and a `content_type`.
  <Expandable>
    <ParamField body="social_account_id" type="string" required>
      UUID of an active social account in the current workspace. Inactive or external accounts are rejected.
    </ParamField>
    <ParamField body="content_type" type="string" required>
      Format for this platform — must match the social account's platform. Discover the catalog with [`GET /content-types`](/api-reference/endpoint/list-content-types).

      Allowed values: `linkedin_post`, `linkedin_carousel`, `linkedin_page_post`, `linkedin_page_carousel`, `x_post`, `facebook_post`, `facebook_reel`, `facebook_story`, `instagram_feed`, `instagram_carousel`, `instagram_reel`, `instagram_story`, `tiktok_video`, `youtube_short`, `threads_post`, `pinterest_pin`, `pinterest_video_pin`, `pinterest_carousel`, `bluesky_post`, `mastodon_post`.
    </ParamField>
    <ParamField body="content" type="string">
      Per-platform text override. Omit to share the same text across platforms (set later via [`PUT /posts/{post}`](/api-reference/endpoint/update-post)). Maximum 63206 characters.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="scheduled_at" type="string">
  ISO 8601 datetime in the future (e.g. `2026-05-10T15:30:00Z`). Required when `status=scheduled`.
</ParamField>

<ParamField body="status" type="string">
  Initial post status. One of:

  - `draft` *(default)* — keep editing
  - `scheduled` — schedule for `scheduled_at`
  - `publishing` — dispatch publish jobs immediately

  To set content, media, and labels in one go after creation, use [`PUT /posts/{post}`](/api-reference/endpoint/update-post). To attach media from URLs, use [`POST /posts/{post}/media`](/api-reference/endpoint/attach-media).
</ParamField>

## Response

Returns `201` with the created post. Response shape is identical to [`GET /posts/{post}`](/api-reference/endpoint/get-post) — see that endpoint for full field documentation.

<RequestExample>
```bash
curl -X POST https://app.trypost.it/api/posts \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "platforms": [
      {
        "social_account_id": "b2c3d4e5-f6a7-8b9c-0d1e-2f3a4b5c6d7e",
        "content_type": "linkedin_post",
        "content": "Hello from the TryPost API!"
      }
    ],
    "scheduled_at": "2026-05-10T15:30:00Z",
    "status": "scheduled"
  }'
```
</RequestExample>
