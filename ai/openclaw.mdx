[OpenClaw](https://openclaw.ai) is an open-source personal AI assistant that runs on your machine and talks across chat channels. You can connect it to TryPost's MCP server so the agent can manage your social media scheduling with natural language.

## Prerequisites

- OpenClaw installed ([install guide](https://docs.openclaw.ai/start/getting-started))
- A TryPost Cloud account with an active trial or subscription ([start a 7-day free trial](/knowledge-base/plans-and-billing))

## Setup

<Steps>
  <Step title="Add the TryPost MCP server">
    Register the remote server with OAuth:

    ```bash
    openclaw mcp add trypost \
      --url https://app.trypost.it/mcp/trypost \
      --transport streamable-http \
      --auth oauth \
      --oauth-scope mcp:use
    ```

    This writes the server under `mcp.servers` in `~/.openclaw/openclaw.json`.
  </Step>
  <Step title="Sign in with OAuth">
    Start the login flow:

    ```bash
    openclaw mcp login trypost
    ```

    OpenClaw prints an authorization URL and stores a temporary PKCE verifier. Open that URL in a browser, sign in to TryPost, and click **Allow**. After approval, pass the returned code back:

    ```bash
    openclaw mcp login trypost --code YOUR_CODE
    ```
  </Step>
  <Step title="Reload and verify">
    Active agent or Gateway processes may still hold the old tool list. Reload MCP clients in this CLI process, or restart the Gateway / publish config from the Control UI:

    ```bash
    openclaw mcp reload
    openclaw mcp doctor trypost --probe
    ```

    Or probe for a live tool list:

    ```bash
    openclaw mcp probe trypost
    ```

    Then ask your OpenClaw agent (Control UI, Telegram, Discord, etc.):

    ```
    List all my scheduled posts
    ```
  </Step>
</Steps>

<Tip>
  You can also manage MCP servers from the Control UI at **Settings → MCP** (`openclaw dashboard`, then `/settings/mcp`). Use **Save & Publish** when editing there so the Gateway picks up changes. Use `openclaw mcp logout trypost` if you need to switch TryPost accounts. If status reports `authorization-required`, run `openclaw mcp login trypost` again.
</Tip>

## Manual configuration

If you prefer to edit the config file directly, this is the shape `openclaw mcp add` writes:

```json5
{
  mcp: {
    servers: {
      trypost: {
        url: "https://app.trypost.it/mcp/trypost",
        transport: "streamable-http",
        auth: "oauth",
        oauth: {
          scope: "mcp:use",
        },
      },
    },
  },
}
```

Save it in `~/.openclaw/openclaw.json`, then run `openclaw mcp login trypost` and complete the `--code` step above.

<Accordion title="Bearer token fallback (CI, headless, no browser)">
  When you can't complete OAuth — CI runners, remote boxes without a browser — use a Personal Access Token instead. Skip `--auth oauth` (static `Authorization` headers are ignored while OAuth auth is enabled) and set a header:

  ```bash
  openclaw mcp add trypost \
    --url https://app.trypost.it/mcp/trypost \
    --transport streamable-http \
    --header "Authorization: Bearer YOUR_API_KEY"
  ```

  Or in config:

  ```json5
  {
    mcp: {
      servers: {
        trypost: {
          url: "https://app.trypost.it/mcp/trypost",
          transport: "streamable-http",
          headers: {
            Authorization: "Bearer ${TRYPOST_API_KEY}",
          },
        },
      },
    },
  }
  ```

  Create the key from **Settings &gt; API Keys** on the [TryPost dashboard](https://app.trypost.it). Prefer an env var over committing the token.
</Accordion>
